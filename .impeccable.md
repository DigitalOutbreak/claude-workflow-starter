# Impeccable — Design Context

> Read this before writing or reviewing UI for this project. Updated after the first design pass — direction pivoted from editorial-bronze to Linear-style after founder feedback.

---

## Design Context

### Users

Developers who already work with AI coding agents — Claude Code, Codex, Cursor, Gemini, Cline, Aider. They visit from npm, GitHub, social, or word-of-mouth. They're on a laptop, probably reading after-hours, in a dark editor with a terminal open. They scan first, decide in 8 seconds whether this is serious, and either keep reading or close the tab.

**Job to be done:** decide whether this scaffold is worth installing, then learn how to actually use it.

### Brand Personality

**Direct, opinionated, useful.** Holds positions instead of hedging. Avoids the SaaS marketing register. Says the concrete thing instead of the smooth abstract thing. Testable claims, not vibes.

### Aesthetic Direction

**Linear-style.** Reference: <https://github.com/voltagent/awesome-design-md/blob/main/design-md/linear.app/DESIGN.md>.

Concretely that means:

- **Dark canvas** `#010102` — near-pure black with a faint blue tint. The deepest dark in modern dev-tool marketing.
- **Four-step surface ladder** — `#0f1011` (surface-1) → `#141516` (surface-2) → `#18191a` (surface-3) → `#191a1b` (surface-4). Hierarchy carried by surface, not by shadow.
- **Single chromatic accent** — Linear lavender-blue `#5e6ad2`. Used sparingly: brand mark, focus rings, primary CTA, link emphasis. Hover `#828fff`, focus `#5e69d1`. No second accent.
- **Hairline borders** — `#23252a` for the standard 1px border on cards and dividers. `#34343a` for stronger.
- **Inter for display and body** — Linear's custom sans is an Inter derivative; Inter is the closest free match. JetBrains Mono for actual code. This is an intentional reference match, not the AI-reflex pick.
- **Negative letter-spacing scales with size** — `-0.03em` at 80px → `-0.02em` at 56px → `-0.01em` at 40px → `0` at body.
- **12px card radii** — never pill, rarely 16px. 16px is reserved for product-screenshot frames.
- **Centered hero**, gridded sections — Linear's marketing pages center the headline and CTAs. We follow.

### Anti-references

These are still rejected, despite the pivot back to Linear style:

- **Gradient text** (background-clip: text). Linear doesn't use it; the impeccable absolute-ban applies.
- **Side-stripe borders** (border-left/right > 1px). Linear doesn't use them; the impeccable absolute-ban applies.
- **Multiple chromatic accents.** Linear's discipline is exactly one accent color. Adding orange, green, pink — even subtly — breaks the system.
- **Atmospheric gradients / spotlight halos.** Linear's marketing does NOT have spotlight gradients on the hero. The surface ladder + hairlines carry depth.
- **Glassmorphism everywhere.** A single backdrop-blur on the sticky nav is fine; pervasive glass effects are not Linear.
- **Pill buttons** (except for tiny status badges). 12px corners are the standard.
- **Editorial section numbering (§01).** That was the wrong vibe. Linear uses short labels and large display headings.

### Design Principles

1. **Match the Linear reference exactly where it makes sense.** Canvas, surface ladder, hairlines, lavender accent, Inter typography, 12px cards. The reference is the spec.
2. **One accent. One.** `#5e6ad2`. If you're tempted to add a second color for variety, find a different solution — surface ladder, type weight, hairline contrast.
3. **Cards are the unit of content presentation.** Feature card, pricing card, testimonial card, product-screenshot card. 12px corners, 1px hairline, surface-1 background. Not glowy. Not gradiented.
4. **Headlines do the work.** Aggressive negative tracking on display sizes (-3px at 80px). Body text at 16-18px with -0.05px tracking. Type carries the page; decoration doesn't.
5. **Product UI is the hero.** Linear's marketing leads with dense, framed product screenshots. We don't have a literal product UI — but the terminal preview is our equivalent and should be treated with the same care: surface-1 panel, 12px or 16px corners, hairline border, mono inside.
